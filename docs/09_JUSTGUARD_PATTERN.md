# 저스트가드 패턴 설계 (Just Guard)

> 보스팀 ↔ 플레이어팀 인터페이스 문서.
> **보스 측 C++는 구현 완료** (아래 §7 참고). 플레이어팀 할 일은 [§6](#6-플레이어팀-구현-항목)에 정리.

---

## 1. 개요

저스트가드는 **기존 AOE 장판 시스템(`ABossPatternActorBase`) 위에 얹는 패턴**입니다.

- 일반 AOE: `CastTime` 동안 빨간 예고 장판 표시 → 시간이 다 되면 장판 안의 플레이어에게 데미지.
- 저스트가드 AOE: 위와 동일하되, **판정 순간에 올바른 타이밍 + 올바른 방향으로 가드(G키)에 성공하면 데미지 무효**, 실패하면 그대로 데미지.

핵심 원칙:

| 원칙 | 내용 |
|---|---|
| 장판 기반 | 예고/판정/도형(원·부채꼴)은 전부 기존 AOE 시스템 재사용 |
| 전조 = 보스 글로우 | 패턴 시작 시 보스가 **노랗게 빛나며**, 이 순간 **모든 플레이어**(장판 위치 무관)에게 저스트가드 1회 기회 부여 |
| 장판 안 = 판정 대상 | **장판이 다 차오른 순간 장판 안에 있던 플레이어만** 가드 판정에 들어감. 장판 밖 플레이어는 판정 로직과 무관 (가드 실패해도 아무 일 없음) |
| 1회성 가드 | 패턴당 **딱 한 번**만 가드 가능. 패턴이 끝나면 기회 회수 |
| 이중 판정 | ① 타이밍(개발자가 지정한 윈도우 안에 눌렀나) + ② 방향(장판 중심을 바라보고 눌렀나) 둘 다 만족해야 성공 |
| 개발자 커스텀 | 성공 윈도우 시작(엇박)/길이/선입력 유예/방향 허용각 전부 패턴별 조정 |
| 키 고정 | 저스트가드 키는 **G키 고정** |

## 2. 예고 연출 — 두 가지 변형

보스 글로우는 **항상** 나오고, 예고 장판은 패턴에 따라 보여줄 수도, 숨길 수도 있습니다.

| 변형 | 흐름 | 장판 BP 설정 |
|---|---|---|
| **A. 장판 보임** | 글로우 → 장판이 **중앙에서부터 점차 차오름** → 다 찼을 때 가드 | `bInstant=false`, `bTelegraphFill=true` |
| **B. 장판 안 보임** | 글로우 → (장판 invisible) → 보스 모션/글로우만 보고 가드 | `bInstant=true` |

- **차오르는 연출 (A)**: 일반 AOE는 전체 범위가 한 번에 빨갛게 표시되지만, 저스트가드는 타이밍이 핵심이므로 **시전 중앙에서 판정 범위 끝까지 `CastTime` 동안 점차 차오르게** 표시합니다. "다 차는 순간 = 판정 기준 시각"을 눈으로 읽을 수 있게. (`bTelegraphFill` — 베이스 장판 공통 옵션으로 구현됨)
- **invisible (B)**: `bInstant`는 이 코드베이스에서 "즉시 히트"가 아니라 **"지면 예고 메시만 숨기고 `CastTime` 판정 타이밍은 그대로 유지"** 입니다. 히트가 앞당겨지지 않습니다.
- **장판 소환 시점**: default는 **보스 글로우와 동시에 바로 소환** (몽타주에서 창 노티파이 시작과 같은 프레임에 스폰 노티파이 배치). 연출에 따라 글로우 이후로 늦출 수도 있습니다.

**보스 노란 글로우**는 저스트가드 창이 열릴 때 자동으로 신호가 나갑니다. BP에서 둘 중 하나에 연출을 바인딩하세요.

- 보스 ASC의 `State.Boss.JustGuardable` 태그 (창 열림 동안 부여)
- `UBossJustGuardComponent::OnJustGuardWindowChanged(bool bOpen)` 델리게이트

## 3. 패턴 흐름 (타임라인)

```
글로우          장판 소환            장판이 차오르는 시간(CastTime)        다 참(T)    판정 시각(J)
  │               │                                                      │            │
  ▼               ▼                                                      ▼            ▼
──┬───────────────┬──────────────────────────────────────────────────────┬────────────┬──────
  │               │              ███ 중앙에서 점차 차오름 ███              │← Judgment →│
  │               │                                                      │   Delay    │
  │  (default: 글로우와 동시 소환.                                 ┌─────────────────┐│
  │   invisible 변형이면 이 구간 내내 안 보임)                      │ 성공 윈도우(파랑) ││
  │                                                               │ [J−Duration, J] ││
모든 플레이어에게                                                  └─────────────────┘│
저스트가드 1회 상태 부여                                     ┌─ T 순간 장판 안에 있던    │
(장판 위치 무관)                                             │  플레이어만 판정 대상 확정 │
                                                                          판정+데미지 확정 = J

패턴 종료(창 닫힘): 모든 플레이어의 1회 상태 회수
```

- **판정 시각(초록선) J** = `T + JudgmentDelay`. **디폴트는 장판이 다 차오른 순간(T) 그대로**이고, 개발자가 엇박을 위해 뒤로 미룰 수 있음. 판정과 데미지 확정이 모두 J에서 일어남.
- **성공 윈도우(파란 박스)** = `[J − GuardWindowDuration, J]`. **항상 판정 시각 직전에 붙어서 끝나는** 구간 — 이 안에 G를 눌러야 성공.

| 케이스 | 성공 윈도우 | 체감 |
|---|---|---|
| `JudgmentDelay = 0` (디폴트) | `[T − Duration, T]` | 장판이 **다 차기 직전 ~ 다 차는 순간**에 가드. "미리 눌러도 되는 유예"는 파란 박스 길이가 그 역할 |
| `JudgmentDelay > 0` (엇박) | `[T + Delay − Duration, T + Delay]` | 장판이 다 차서 **사라진 뒤**, 보스 모션을 보고 엇박에 가드 |

- 판정 대상(= T 순간 장판 안에 있던 플레이어)이 윈도우 안에 G를 눌렀고 방향도 통과 → **성공 (데미지 무효)**
- 윈도우 밖 / 방향 틀림 / 미입력 → **실패 (J에 데미지 적용)**
- **장판 밖 플레이어** → 판정 자체가 없음. G를 눌러 기회를 소모했더라도 아무 일 없음.
- 가드 입력은 **패턴당 1회만 유효.** 너무 일찍 눌러 실패하면 그 패턴에서 다시 못 누름.

> **판정 대상 스냅샷**: 판정 대상은 **T 순간의 위치**로 확정됩니다. T 이후 장판 밖으로 빠져나가도 판정 대상에서 빠지지 않습니다 (차오름을 보고 들어왔다면 가드로 막아야 함).

### 타이밍 파라미터 (장판의 `저스트가드 (Just Guard)` 이펙트에서 편집)

| 파라미터 | 기본값 | 의미 |
|---|---|---|
| `JudgmentDelay` | **0** | 판정 시각(초록선)을 T보다 뒤로 미는 엇박 오프셋. `J = T + Delay`. 0 = 다 차는 순간이 곧 판정 |
| `GuardWindowDuration` | 0.3 | 성공 윈도우(파란 박스) 길이. 윈도우는 항상 `[J − Duration, J]` |
| `GuardAngleTolerance` | 60 | 방향 판정 허용 각도(도) — §4 |

## 4. 방향 판정

저스트가드 장판의 **중심부**를 바라보고 눌러야 합니다.

1. 플레이어는 장판 중심부에 **마우스 커서**를 대고 G키를 누른다.
2. 캐릭터가 커서 방향(= 장판 중심 방향)을 바라보며 **저스트가드 모션**을 취한다.
3. 서버가 "가드 순간 캐릭터 정면"과 "캐릭터 → 장판 중심 방향" 사이 각도를 비교, `GuardAngleTolerance` 이내면 통과.

> 방향의 기준점(장판 중심)은 보스 쪽 장판이 알고 있습니다. 플레이어 쪽은 **커서 방향으로 캐릭터를 회전시킨 뒤 가드 이벤트만 보내면** 됩니다. 보스는 이벤트의 Instigator(=플레이어)의 **전방 벡터**를 방향으로 읽으므로, 이벤트 페이로드에 방향을 실을 필요가 없습니다.

## 5. 결과 & GAS 태그

| 결과 | 처리 |
|---|---|
| **성공** (타이밍 ✅ + 방향 ✅) | 데미지 무효. 보스 ASC에 `State.Boss.PatternResult.JustGuarded` 부여 → 패턴 분기(Branch)에서 활용. 성공 연출은 플레이어팀 |
| **실패** (타이밍 ❌ / 방향 ❌ / 미입력) | 기존 AOE와 동일하게 데미지 GE 적용 |
| **장판 밖** | 판정 없음 (성공도 실패도 아님) |

`PatternResult.JustGuarded`는 카운터와 동일하게 **패턴 어빌리티 종료 시 자동 정리**됩니다.

### 태그 목록 (전부 `BossGameplayTags`에 네이티브 정의 완료)

| 태그 | 부여 대상 | 수명 |
|---|---|---|
| `State.Boss.JustGuardable` | 보스 | 창 열림 동안. **보스 노란 글로우를 여기에 바인딩** |
| `State.Player.GuardReady` | **모든** 플레이어 (장판 위치 무관) | 창 열림(글로우) 시 부여(복제됨) → G 입력 소모 또는 패턴 종료 시 회수. **이 태그가 있어야 가드 발동 가능** |
| `State.Player.Guarding` | 플레이어 | 가드 모션 재생 중 (부여/제거는 **플레이어 GA가 관리**, 이동/스킬 잠금 바인딩용) |
| `State.Boss.PatternResult.JustGuarded` | 보스 | 가드 성공 시 부여, 패턴 종료 시 자동 제거 (분기용) |
| `Event.Boss.JustGuardInput` | 보스로 전송 | 플레이어가 G 발동 시 보내는 GameplayEvent (Instigator=플레이어) |

**"딱 한 번" 규칙**: `GuardReady`는 가드 입력 순간 소모(제거)됩니다. 성공/실패/장판 밖 무관하게 1패턴 1가드.

## 6. 플레이어팀 구현 항목

### 6-1. 입력
- [ ] **G키** 입력 액션 (Enhanced Input). 키는 G 고정.
- [ ] G키 → 저스트가드 어빌리티(`GA_JustGuard`) 발동.

### 6-2. GA_JustGuard (플레이어 어빌리티)
- [ ] **발동 조건**: 소유자에게 `State.Player.GuardReady` 가 있을 때만 (`ActivationRequiredTags`).
- [ ] **마우스 커서 방향으로 캐릭터 회전** 후 저스트가드 모션(몽타주) 재생.
- [ ] 회전 후 보스에게 `SendGameplayEventToActor(Boss, Event.Boss.JustGuardInput, Instigator=자기자신)` 전송. **성공/실패 판정·GuardReady 소모는 전부 보스(서버)가 처리** — 플레이어는 보고만.
- [ ] 가드 모션 중 `State.Player.Guarding` 부여 (이동/스킬 잠금은 팀 정책대로).

### 6-3. 연출
- [ ] 저스트가드 **모션(가드 자세)** 몽타주.
- [ ] 저스트가드 **성공 연출** (판정 결과는 서버 태그/이벤트로 내려옴).
- [ ] (선택) `State.Player.GuardReady` 태그 리슨 → "지금 가드 가능" UI.

### 6-4. 하지 않아도 되는 것 (보스팀이 이미 구현)
- 장판 스폰/차오르는 예고/도형 판정, 타이밍 윈도우 계산, 방향 각도 판정
- `GuardReady`/`JustGuardable` 태그 부여·회수, 보스 글로우 신호 발신
- 성공 시 데미지 무효, 실패 시 데미지 적용, 장판 밖 무판정

### 애니메이터 배치 계약 (중요)
- 저스트가드 창 노티파이(`Boss Just Guard Window`) 구간은 **글로우 시작 ~ 판정 시각 J(`T + JudgmentDelay`)** 까지 덮어야 합니다. 창이 닫히면 `GuardReady`가 회수되므로, 엇박(`JudgmentDelay`) 패턴에서는 구간을 T 너머까지 넉넉히 잡으세요.
- 장판 스폰 노티파이는 **default로 창 노티파이 시작과 같은 프레임**에 배치(= 글로우와 동시 소환). 연출상 늦춰도 됩니다.

### 남은 연동 (실제 스킬 붙일 때)
- 지금은 임시 디버그 **G키**로 검증합니다(§8). 실제로는 플레이어 `GA_JustGuard`가 `Event.Boss.JustGuardInput`를 보내면 동일 경로로 판정됩니다.
- 방향 판정은 플레이어가 커서 방향으로 회전한 뒤 이벤트를 보내야 정상 동작합니다.

## 8. 디버그 (임시)

카운터의 **Q키** 강제 카운터와 같은 방식으로, **G키** 디버그를 넣어 두었습니다 ([LostArkPlayerController](../LostArk/Source/LostArk/Core/LostArkPlayerController.cpp), 나중에 삭제 예정).

| 상황 | G키를 누르면 |
|---|---|
| 저스트가드 패턴이 아닐 때(GuardReady 없음) | 화면에 **"저스트가드 모션 비활성화"** |
| 저스트가드 패턴 중(GuardReady 보유) | **"저스트가드 모션 발동"** → 판정 시각 J에 판정 대상별로 **"저스트가드 성공"**(초록) / **"실패 (타이밍/방향/미입력)"**(빨강) 표시. 장판 밖이면 아무 표시 없음 |

- 디버그 G는 캐릭터 회전 없이 현재 정면으로 이벤트를 보내므로, **방향 판정 때문에 실패**할 수 있습니다. 타이밍만 검증하려면 장판의 `저스트가드 (Just Guard)` 이펙트에서 `bDebugBypassDirection = true` 로 두거나, 보스를 바라본 채 누르세요.
- 성공/실패 메시지는 서버(리슨 서버) 화면에 표시됩니다.
